Lanjutkan project Wed Invitation Hub yang sudah ada. Jangan rewrite project dari nol. Jangan merusak fitur yang sudah berjalan seperti auth, invitations, template builder, RSVP, wishes, pricing, billing, dan admin dashboard.

Fokus sekarang hanya pada penyempurnaan sistem pembayaran transfer manual agar benar-benar profesional, stabil, dan berjalan end-to-end.

TUJUAN UTAMA
Sistem pembayaran harus mendukung flow lengkap berikut:

USER:
1. memilih paket Premium atau Pro
2. sistem membuat order
3. user melihat instruksi transfer
4. user upload bukti transfer
5. status pembayaran menjadi menunggu konfirmasi
6. user menerima notifikasi status pembayaran
7. jika pembayaran disetujui, paket user aktif otomatis
8. user menerima email konfirmasi dan invoice / ringkasan pembayaran

ADMIN:
9. admin melihat daftar pembayaran masuk
10. admin melihat detail pembayaran dan bukti transfer
11. admin bisa approve / reject
12. admin bisa menandai pembayaran selesai atau belum selesai
13. admin bisa mengatur rekening tujuan transfer
14. admin bisa mengatur catatan pembayaran

Jangan buat UI dummy.
Semua tombol, form, route, status, email, dan perubahan paket harus benar-benar aktif.

==================================================
1. PASTIKAN FLOW PEMBAYARAN BERJALAN
==================================================

Saat user memilih paket berbayar:
- buat order baru
- payment_method = bank_transfer
- payment_status = pending
- order_number unik
- amount sesuai harga paket
- subscription user dibuat / diupdate menjadi pending_payment

Saat user upload bukti transfer:
- simpan data konfirmasi transfer
- simpan gambar bukti transfer
- ubah payment_status menjadi waiting_confirmation

Saat admin approve:
- ubah payment_status menjadi paid
- ubah subscription user menjadi active
- aktifkan plan user sesuai paket yang dibeli
- isi started_at
- nonaktifkan paket aktif lama jika perlu

Saat admin reject:
- ubah payment_status menjadi rejected
- subscription jangan aktif
- user dapat informasi bahwa pembayaran ditolak

==================================================
2. TAMBAHKAN STATUS "SELESAI ATAU BELUM"
==================================================

Selain payment_status, tambahkan status progress admin yang jelas.

Gunakan salah satu pendekatan yang rapi:

Option A:
Tambahkan field order_fulfillment_status:
- not_reviewed
- reviewing
- completed
- rejected

Option B:
Gunakan order_status terpisah:
- pending
- waiting_confirmation
- under_review
- completed
- rejected

Yang penting:
- admin bisa menandai pembayaran sudah selesai / belum selesai
- user bisa melihat status yang jelas
- status ini tersimpan di database dan tampil di dashboard
