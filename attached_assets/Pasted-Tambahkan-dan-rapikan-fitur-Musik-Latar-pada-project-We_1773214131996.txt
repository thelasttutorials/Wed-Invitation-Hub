Tambahkan dan rapikan fitur Musik Latar pada project Wed Invitation Hub agar admin dan user dapat mengatur musik undangan dengan tampilan dan flow seperti contoh UI yang sudah ada.

Jangan rewrite project dari nol.
Jangan merusak fitur lain seperti auth, pricing, billing, template builder, invitation, RSVP, wishes, dan admin dashboard.
Fokus hanya pada fitur musik latar untuk template/undangan.

TUJUAN UTAMA
Saya ingin ada fitur Musik Latar yang lengkap seperti ini:

ADMIN / USER bisa:
1. memilih musik dari perpustakaan lagu
2. memasukkan URL musik custom
3. memberi label lagu
4. mempreview lagu sebelum dipakai
5. menyimpan musik untuk template atau invitation
6. memilih apakah musik autoplay, loop, dan volume default
7. menggunakan musik dari:
   - MP3 direct URL
   - YouTube
   - Spotify
8. halaman undangan publik memutar musik yang dipilih jika diaktifkan

Jangan buat UI dummy.
Semua input, tombol, preview, save, dan player harus benar-benar aktif.

==================================================
STEP 1 — UI MUSIK LATAR
==================================================

Buat / rapikan panel Musik Latar dengan tampilan seperti contoh screenshot.

Komponen UI minimal:

- title: Musik Latar
- description:
  Pilih dari perpustakaan atau masukkan URL MP3 sendiri

Bagian 1:
Pilih dari Perpustakaan

Tampilkan daftar lagu dalam panel scrollable:
- thumbnail
- judul lagu
- artis
- klik untuk memilih

Contoh item:
- Marry You — Bruno Mars
- Perfect — Ed Sheeran
- Sempurna — Andra & The Backbone
- Thinking Out Loud — Ed Sheeran

Bagian 2:
URL Musik Kustom

Input:
- URL Musik Kustom (MP3 atau YouTube atau Spotify)

Bagian 3:
Label Lagu (opsional)

Input:
- label lagu

Bagian 4:
Preview / status pilihan

Tampilkan misalnya:
- Dipilih dari perpustakaan: Perfect
atau
- URL custom aktif

Bagian 5:
Pengaturan tambahan
- autoplay on/off
- loop on/off
- volume default (0 sampai 100)
- start muted on/off jika perlu

==================================================
STEP 2 — SUMBER MUSIK YANG DIDUKUNG
==================================================

Dukung sumber musik berikut:

1. Library internal
- daftar lagu yang disediakan sistem
- tiap lagu punya title, artist, thumbnail, source_url, source_type

2. MP3 direct URL
- URL file .mp3 yang langsung bisa diputar

3. YouTube
- URL youtube watch / short / embed
- ekstrak video id jika perlu
- gunakan embed/player yang aman dan stabil

4. Spotify
- dukung URL track Spotify resmi
- gunakan embed/preview resmi jika memungkinkan
- jangan mengklaim bisa memutar full audio Spotify jika platform/embed tidak mengizinkan
- jika keterbatasan teknis ada, gunakan embed preview resmi dan jelaskan di implementasi

Sumber type enum:
- library
- mp3
- youtube
- spotify

==================================================
STEP 6 — VALIDASI URL MUSIK
==================================================

Tambahkan validasi sumber musik.

MP3:
- URL harus valid
- jika memungkinkan cek ekstensi atau content type

YouTube:
- dukung format:
  - youtube.com/watch?v=
  - youtu.be/
  - youtube.com/embed/
- ekstrak video id dengan aman

Spotify:
- dukung format track Spotify resmi
- ekstrak track id jika perlu
- validasi domain spotify

Jika URL tidak valid:
- tampilkan pesan error yang jelas
- jangan simpan data rusak

==================================================
STEP 7 — PREVIEW MUSIK
==================================================

Tambahkan preview player di panel musik.

Jika library atau mp3:
- gunakan audio player HTML5 atau player custom ringan

Jika YouTube:
- gunakan embedded preview / iframe / player yang sesuai
- preview harus aman dan tidak merusak layout

Jika Spotify:
- gunakan embed preview resmi

Pastikan preview:
- bisa diputar
- bisa dihentikan
- tidak error jika source invalid
- tidak bentrok dengan player publik invitation

==================================================
STEP 8 — PLAYER DI HALAMAN UNDANGAN PUBLIK
==================================================

Halaman undangan publik harus memakai musik yang telah dipilih.

Fitur player publik:
- tombol play/pause
- indikator judul lagu jika ada
- volume default
- loop jika diaktifkan
- autoplay jika diaktifkan dan browser mengizinkan
- fallback jika autoplay diblokir browser

Catatan penting:
- browser sering memblokir autoplay audio
- jika autoplay gagal, tampilkan tombol play yang jelas
- jangan membuat halaman error hanya karena autoplay diblokir

Jika source:
MP3:
- gunakan audio player native/custom

YouTube:
- gunakan embed/player yang sesuai

Spotify:
- gunakan embed resmi bila didukung

==================================================
STEP 12 — ERROR PREVENTION
==================================================

Cegah error berikut:
- URL tidak valid
- preview gagal membuat halaman crash
- thumbnail kosong membuat layout rusak
- source_type mismatch
- autoplay diblokir browser
- iframe YouTube/Spotify merusak responsivitas
- state musik tidak tersimpan
- duplicate player berjalan bersamaan
- broken import
- null data crash

Tambahkan fallback aman untuk:
- thumbnail kosong
- label kosong
- source_url kosong
- source invalid
