Lanjutkan project Wed Invitation Hub yang sudah ada. Jangan rewrite project dari nol. Jangan merusak fitur yang sudah berjalan seperti landing page, admin login, invitations, RSVP, wishes, dan struktur folder client/server/shared.

Tujuan utama adalah menjadikan project ini platform undangan digital pernikahan SaaS dengan:

1. login/register menggunakan email OTP
2. paket langganan Gratis / Premium / Pro
3. pembayaran paket Premium/Pro menggunakan transfer bank manual
4. upload bukti transfer oleh user
5. admin review pembayaran dan approve / reject
6. fitur undangan aktif atau terkunci sesuai paket user
7. semua data tersimpan di database
8. tidak ada UI dummy atau route mati

Kerjakan perubahan secara bertahap dan pastikan server tetap bisa dijalankan setelah perubahan.

==================================================
STEP 1 — USER AUTH EMAIL OTP
==================================================

Tambahkan auth user berbasis email OTP tanpa password.

Flow:

1 user buka halaman login/register
2 user memasukkan email
3 user klik "Kirim Kode"
4 server generate OTP 6 digit
5 OTP dikirim ke email user
6 user memasukkan kode OTP
7 jika valid user login

Tambahkan tabel:

users
- id
- email
- created_at
- updated_at
- is_verified

email_verifications
- id
- email
- code
- expires_at
- used_at
- created_at

Aturan OTP:

- kode 6 digit
- berlaku 5 menit
- hanya bisa dipakai sekali
- rate limit request OTP

API:

POST /api/auth/request-code
POST /api/auth/verify-code
GET /api/auth/me
POST /api/auth/logout

Frontend:

/login
/register

UI:

step 1 input email
step 2 input OTP
