Lakukan audit menyeluruh pada project Wed Invitation Hub yang sudah ada, lalu perbaiki semua bagian yang belum berjalan, belum tersambung, atau script-nya belum lengkap.

Jangan rewrite project dari nol.
Jangan hapus fitur yang sudah bekerja.
Jangan membuat UI dummy.
Fokus pada audit teknis penuh lalu perbaikan end-to-end sampai project stabil.

TUJUAN UTAMA
1. cek seluruh project
2. temukan semua script yang belum ada atau belum lengkap
3. temukan semua route yang mati
4. temukan semua tombol yang tidak bekerja
5. temukan semua integrasi frontend-backend-database yang putus
6. perbaiki semuanya secara bertahap
7. pastikan project bisa dijalankan tanpa kendala besar

==================================================
STEP 2 — IDENTIFIKASI MASALAH
==================================================

Temukan dan kelompokkan semua masalah ke kategori ini:

WORKING
- bagian yang sudah benar-benar berjalan

BROKEN
- ada tetapi error / tidak jalan

MISSING
- dibutuhkan tetapi belum ada implementasi

HALF-DONE
- sudah ada UI atau route tapi backend/script belum lengkap

UNUSED / DEAD CODE
- kode tidak terpakai, import rusak, route tidak dipakai, komponen yatim

==================================================
STEP 5 — PERBAIKI ROUTE DAN TOMBOL MATI
==================================================

Audit semua tombol, link, dan navigasi.

Pastikan tidak ada:
- card template yang tidak clickable
- tombol demo mati
- tombol pakai tema mati
- link privacy / terms / contact mati
- menu dashboard yang blank
- submit form yang tidak memanggil API
- action approve/reject yang tidak mengubah status

Setiap tombol harus:
- punya action valid
- tersambung ke route atau API yang benar
- ada feedback sukses/gagal

INSTRUKSI TAMBAHAN
- Kerjakan audit dulu, lalu perbaiki bertahap
- Jangan merusak fitur lain

- Pastikan backend, database, dan integrasi ikut diperbaiki
- Prioritas utama adalah stabilitas project dan semua fitur utama berjalan end-to-end


Kerjakan hanya berdasarkan hasil audit nyata pada project ini.
