Lanjutkan project ini menjadi platform undangan digital pernikahan dengan sistem paket berlangganan dan aktivasi fitur berdasarkan paket. Gunakan pembayaran transfer bank manual saja. Jangan bangun ulang dari nol. Lanjutkan dari struktur project yang sudah ada.

TUJUAN
Buat sistem paket:
1. Mulai Gratis
2. Premium
3. Pro

Setiap user bisa memilih paket.
Fitur yang aktif harus mengikuti paket yang dipilih.
Untuk paket berbayar, aktivasi dilakukan setelah user melakukan pembayaran transfer dan admin mengonfirmasi pembayaran.

ATURAN PENTING
- Jangan merusak landing page, admin, invitation, RSVP, dan auth yang sudah ada.
- Jangan buat payment gateway otomatis.
- Gunakan pembayaran transfer manual saja.
- Semua data harus tersimpan di database.
- Semua status paket dan pembayaran harus bisa dikelola dari admin.
- Jangan buat UI dummy. Semua tombol dan status harus benar-benar berfungsi.

==================================================
1. PAKET LANGGANAN
==================================================

Buat 3 paket:

A. Mulai Gratis
- harga: 0
- maksimal 1 undangan
- tema standar saja
- RSVP aktif
- wishes aktif
- galeri maksimal 3 foto
- tanpa background music premium
- tanpa custom domain
- tanpa amplop digital

B. Premium
- harga bisa diedit admin
- maksimal 5 undangan
- semua tema standar + beberapa tema premium
- RSVP aktif
- wishes aktif
- galeri maksimal 15 foto
- background music aktif
- love story aktif
- gift/amplop digital aktif
- prioritas support

C. Pro
- harga bisa diedit admin
- undangan tidak terbatas atau limit tinggi
- semua tema premium aktif
- RSVP aktif
- wishes aktif
- galeri penuh
- background music aktif
- love story aktif
- gift/amplop digital aktif
- custom domain opsional
- fitur paling lengkap

==================================================
4. FLOW USER
==================================================

Flow yang diinginkan:

1. User daftar / login
2. User masuk ke halaman paket /pricing atau /dashboard/billing
3. User bisa pilih:
   - Mulai Gratis
   - Premium
   - Pro
4. Jika pilih Mulai Gratis:
   - paket langsung aktif
   - user bisa langsung membuat undangan sesuai limit paket gratis
5. Jika pilih Premium atau Pro:
   - sistem membuat order baru
   - tampilkan instruksi transfer bank
   - user upload bukti transfer
   - status menjadi waiting_confirmation
   - admin meninjau pembayaran
   - jika admin setujui, paket aktif
   - jika admin tolak, status rejected

==================================================
8. AKTIVASI FITUR BERDASARKAN PAKET
==================================================

Pastikan fitur benar-benar dibatasi sesuai paket.

Contoh:
- jika paket gratis, user tidak bisa pilih template premium
- jika paket gratis, user tidak bisa upload galeri lebih dari batas
- jika paket gratis, field gift/amplop digital nonaktif
- jika paket belum active, fitur premium tidak bisa digunakan
- jika paket Premium active, fitur premium aktif sesuai haknya
- jika paket Pro active, semua fitur aktif

Pembatasan harus dilakukan:
1. di frontend UI
2. di backend validation

Jangan hanya sembunyikan tombol. Backend juga harus menolak jika user tidak berhak.

10. ADMIN KONFIRMASI PEMBAYARAN
==================================================

Admin harus punya halaman untuk review pembayaran.

Tampilkan:
- nama user
- email user
- paket yang dibeli
- nominal
- bukti transfer
- tanggal upload
- catatan user

Admin action:
- Approve
- Reject

Jika Approve:
- order payment_status jadi paid
- subscription user jadi active
- paket user berubah sesuai plan

Jika Reject:
- order payment_status jadi rejected
- subscription tidak aktif
- user bisa upload ulang bukti jika diperlukan
